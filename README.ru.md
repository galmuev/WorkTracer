Локальный трекер рабочего времени для Windows. WorkTracer определяет активную программу, распределяет время по проектам и хранит всю статистику только на компьютере пользователя.

## Возможности

- Автоматический учёт времени по активному окну.
- Пауза при бездействии пользователя и ручная остановка учёта.
- Отдельная статистика по программам и проектам.
- Обратимое исключение служебных окон и проектов, время которых учитывать не нужно.
- Объединение связанных файлов и проектов из разных программ в именованные контейнеры.
- Связывание программ без доступного имени документа, например ZBrush, с выбранным рабочим проектом.
- Расчёт стоимости времени по заданной почасовой ставке.
- Работа в системном трее.
- Автозапуск вместе с Windows непосредственно в системном трее.
- Русский и английский интерфейс.

## Как определяется проект

Для каждой программы можно выбрать подходящий режим:

- **Первая часть заголовка** — имя файла извлекается из заголовка окна по указанным расширениям.
- **Выбранная часть заголовка** — проектом становится выбранный фрагмент заголовка или название рабочей папки.
- **Отслеживаемый файл** — пользователь добавляет файлы вручную, а активным считается последний сохранённый файл. Этот метод может быть ненадёжным.
- **Вся программа** — всё время записывается в один проект с названием программы.

Режим **Вся программа** поддерживает связи. У программы может быть несколько назначений, но одновременно активным будет только одно. При работе в программе время сохраняется в её собственной статистике и дополнительно направляется в выбранный проект.

## Начало работы

1. Запустите установщик или portable-версию WorkTracer.
2. Откройте раздел **Программы** и добавьте нужное приложение по EXE-файлу, имени процесса или с помощью кнопки **Активное окно**.
3. Выберите способ определения проекта и при необходимости укажите расширения файлов.
4. Откройте рабочую программу — учёт времени начнётся автоматически.

Закрытие основного окна сворачивает WorkTracer в системный трей. Для полного завершения работы выберите **Выход** в контекстном меню значка.

## Данные и конфиденциальность

WorkTracer не отправляет статистику в интернет. Названия приложений, проектов, отслеживаемые файлы и интервалы времени сохраняются в локальной базе SQLite в каталоге профиля пользователя.

База использует foreign keys, WAL и транзакции. Открыть папку данных и создать проверенную резервную копию можно в разделе **Настройки**.

Автоматические резервные копии создаются после изменений с RPO до 15 минут. Хранятся десять последних поколений резервных копий и ежедневные копии за семь дней. Перед очисткой данных и массовым удалением создаётся отдельная проверенная копия.

Определение активного окна поддерживается только в Windows. Для установленной или portable-версии отдельная установка Node.js не требуется.

## Запуск из исходников

```powershell
npm install
npm start
```

Сборка для Windows:

```powershell
npm run pack:win       # папка release/win-unpacked
npm run portable:win   # portable EXE
npm run dist:win       # установщик NSIS
```

## Технологии

Electron, Node.js, SQLite (`better-sqlite3`), Tailwind CSS, Windows PowerShell и Win32 API.

SQLite является единственным источником рабочих данных приложения. База хранится в каталоге профиля пользователя и использует foreign keys, WAL, транзакции, проверяемые многопоколенные резервные копии и автоматическое восстановление после повреждения.

Архитектура хранилища и восстановления описана в `docs/sqlite-persistence-design.md`.

Проверки проекта запускаются командой `npm run quality`; production-сборка создаётся командой `npm run pack:win`.
